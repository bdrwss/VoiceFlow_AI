模块采用单入口 React 应用（`src/main.tsx` → `App`），通过 Tauri Webview 与 Rust 后端通信：
- 窗口层：`App` 根据 `getCurrentWindow().label` 区分主窗口与 label=`indicator` 的小药丸浮窗，后者仅渲染极简状态条并通过 `WebviewWindow.emit` 双向事件同步。
- 业务编排集中在 `App.tsx`：监听 Rust 全局快捷键事件 `shortcut-state`，驱动 `startRecording` / `stopAndProcess` 流程；同时维护 `initializing→idle→recording→transcribing→rewriting→success/error` 状态机，并控制浮窗显隐与定位。
- 组件层：`components/MainPanel`、`HistoryPanel`、`SettingsPanel` 为无状态展示组件，由 `App` 注入状态与回调。
- 数据持久化层：`hooks/useSettings.ts` 与 `hooks/useHistory.ts` 封装 `localStorage` 读写，前者还通过 `invoke('set_listen_key')` 同步至 Rust。
- 工具层：`utils/audio.ts` 封装 `AudioRecorder`（`getUserMedia` + `AudioWorkletNode` 子线程采集，内置 RMS VAD 静音切除与 `float32ToWav` 转换）、`utils/whisper.ts`（Xenova Whisper 推理）、`utils/api_asr.ts`（Groq/OpenAI 兼容流式 API）、`utils/llm.ts`（DeepSeek 等 LLM 文本润色）。
- 音频工作线程：`public/recorder-worklet.js` 作为独立 AudioWorkletProcessor 以 4096 采样块向主线程推送 Float32Array，避免主线程卡顿。
依赖方向单向：App → hooks/utils/components，组件不直接调用 Tauri invoke，仅通过 props 暴露回调。
