根目录作为 Tauri 工程入口，src-tauri/ 中的 Cargo.toml 与 tauri.conf.json 定义双窗口：label=main 的透明无边框主窗口加载 index.html，label=indicator 的胶囊窗口加载 pill.html；Vite 通过 beforeDevCommand/beforeBuildCommand 与 frontendDist 被 Tauri 驱动，形成 npm run tauri dev/build 的统一工作流。
- 前后端通信：前端通过 @tauri-apps/api invoke 调用 Rust 侧命令（src-tauri/src/main.rs 注册），Rust 侧再桥接系统 API（剪贴板、键盘模拟、全局快捷键）和本地 SenseVoice ONNX 模型（src-tauri/src/sensevoice.rs）。
- 版本同步：package.json 的 version 字段由 scripts/sync-version.cjs 在发布前自动注入到 src-tauri/Cargo.toml 与 tauri.conf.json，保证 JS/Rust 两侧版本号一致。
- 安全边界：tauri.conf.json 的 security.csp 与 capabilities/*.json 共同约束前端可访问的后端命令与网络源，Rust 侧仅暴露必要 IPC 接口。
