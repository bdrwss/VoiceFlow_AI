# VoiceFlow AI (语音随写·智能听写助手) 演进路线图

## 1. 当前完成度分析

目前项目已经高度对齐了初始 PRD，核心骨架和主要交互均已实现并跑通。

### ✅ 已完成功能
1. **核心转写流**：
   - 快捷键按下录音、松开自动提交处理的闭环。
   - 支持云端大模型 API 识别与本地端侧推理（SenseVoice/Whisper）。
2. **AI 润色与替换**：
   - 根据预设 Prompt 风格（如去口语化、书面语等）调用大模型进行文本优化。
   - 通过 Rust 后端 (`enigo`) 模拟退格删除原字并贴入 AI 优化结果。
3. **UI 与系统级交互**：
   - 多面板应用界面（听写、历史、设置）。
   - 沉浸式浮空胶囊窗（Indicator），支持实时麦克风音量波形展示。
   - 依赖 Tauri 实现系统托盘、全局快捷键监听、特定应用黑名单屏蔽。

## 3. 下一步开发计划 (Next Steps)

建议按照以下三个阶段逐步推进，您可以指定从哪个阶段或具体任务开始。

### 阶段一：体验打磨与稳定性修复（高优先级） [已完成]
- [x] **优化文本替换逻辑**：使用 `Shift + 批量左方向键` 选中替代逐字 Backspace 方案，提升长文本替换速度和撤销体验。
- [x] **实现录音超时保护**：在 `App.tsx` 中增加计时器，录音超过 5 分钟自动调用 `stopAndProcess()` 并给用户超时的 Toast 提示。
- [x] **异常与兜底处理**：
  - [x] 增加对麦克风权限被系统拒绝的明确引导。
  - [x] 处理 AI 接口超时重试机制。
- [x] **输入法兼容性防御**：在模拟按键前强制释放修饰键，避免快捷键残留导致的输入法卡死。
- [x] **SenseVoice 模型可视化管理**：提供强制重新下载选项，以防模型损坏。

### 阶段二：功能拓展与高级特性（中优先级） [已完成]
- [x] **场景化 Prompt 一键切换**：在浮空状态窗与主界面，增加快速切换，允许用户随时在“口语、正式、简明、学术”间切换。
- [x] **历史记录功能增强**：支持通过历史记录再次重试（如果某次网络问题导致润色失败，可点击历史记录进行重新润色）。
- [x] **VAD 算法精细化过滤**：引入 50ms 滑动窗口 RMS 能量检测，精确过滤鼠标点击、键盘敲击等瞬时高频噪音并静默拦截。
- [x] **SenseVoice 延迟静默预热**：开机启动后延迟 4 秒执行伪推理，利用 OS Page Cache 将模型预载至内存，解决首次语音识别冷启动延迟。
- [x] **个人专有词库/热词支持**：允许用户配置常用专业术语，传给 ASR 引擎（如果是云端 API 或支持 prompt 的 Whisper）以提高专业场景识别率。
- [x] **录音全局音量压限 (Audio Ducking)**：在按住快捷键说话期间，自动将电脑主音量降低至 5%，避免识别到电脑本身的系统音或背景音乐，松开按键后自动恢复正常音量。

### 阶段三：应用打包、自启配置与发布优化 (✅ 已完成)
*   **基础信息完善**：
    *   **应用图标 (App Icon)**：使用 macOS/Windows 风格的简约麦克风图标，配置 `tauri icon` 生成全平台图标。
    *   **元数据 (Metadata)**：修改 `tauri.conf.json` 和 `Cargo.toml`，更新产品名 (VoiceFlow AI)、版本号 (v1.0.0)、描述等。

*   **版本更新系统 (Updater)**：
    *   引入 `tauri-plugin-updater`。
    *   配置 GitHub Releases 为更新分发源，添加公钥验证。
*   **打包与体积优化 (Bundler)**：
    *   调整 `Cargo.toml` profile (lto, strip) 极限压缩产物体积。
    *   配置 Windows 下的 NSIS 安装包参数，支持一键安装。
*   **CI/CD (GitHub Actions) (可选)**：
    *   编写 `.github/workflows/release.yml`，在打 tag 时自动编译 Windows/macOS 并上传 Release 资产。

### 阶段四：高阶特性与生态拓展（规划中）

1. **极致的隐私与纯离线化 (Local LLM 集成)**
   *目前语音识别 (SenseVoice) 已经实现了离线化，但文本润色仍依赖云端 LLM (如 OpenAI 等)。*
   - **接入本地大模型**：增加对本地大语言模型服务（如 Ollama, LM Studio 提供的本地接口）的支持。
   - **核心价值**：实现真正意义上的 100% 纯离线、零数据外泄，吸引对隐私要求极高的企业用户或敏感岗位群体。

2. **多模态与屏幕感知 (Screen-Aware AI)**
   *将单纯的“语音输入”升级为“结合当前屏幕上下文的智能助手”。*
   - **屏幕截图/文本截取**：在按下录音快捷键时，可选地抓取当前光标所在窗口的文本或对当前屏幕进行截图。
   - **应用场景**：比如选中一封冗长的英文邮件，按住快捷键说“帮我用礼貌的语气回复并拒绝他”，AI 会结合屏幕内容和语音指令直接输出回复。

3. **插件化与工作流生态 (Workflow Integration)**
   *从单纯的“打字工具”演变为“生产力桥梁”。*
   - **目标应用直连**：针对高频生产力工具（Notion、Obsidian、Word、飞书等）开发专属的快捷输出适配。
   - **自定义 Webhook/API 触发**：允许语音指令触发预设的网络请求，例如“添加一条待办事项：下午三点开会”，直接对接到用户的待办事项软件中。

4. **国际化与实时翻译 (i18n & Real-time Translation)**
   - **多语言混合识别**：针对中英夹杂等场景进一步优化 ASR。
   - **语音输入，外语输出**：支持“说中文，直接上屏英文（或其他外语）”的功能，极其适合跨境电商或跨国远程工作者。
   - **软件界面国际化**：为应用前端增加多语言 (i18n) 支持（如中、英、日等），为产品的海外推广做准备。

5. **跨端数据同步 (Cloud Sync)**
   - **引入用户系统或第三方同步**：引入用户系统（或者基于 iCloud/WebDAV 等简单的第三方网盘同步机制）。
   - **配置漫游**：允许用户在不同设备间同步他们的“自定义 Prompt”、“专有词库/热词”以及“系统设置”。
