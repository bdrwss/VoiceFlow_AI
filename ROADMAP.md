# VoiceFlow AI (语音随写·智能听写助手) 演进路线图

## 1. 当前完成度分析

目前项目已经高度对齐了初始 PRD，核心骨架和主要交互均已实现并跑通。

### ✅ 已完成功能
1. **核心转写流**：
   - 快捷键按下录音、松开自动提交处理的闭环。
   - 支持云端大模型 API 识别与本地端侧推理（SenseVoice/Whisper）。
2. **AI 润色与替换**：
   - 根据预设 Prompt 风格（如去口语化、书面语等）调用大模型进行文本优化。
   - 通过 Rust 后端 (`enigo`) 模拟退格删除原字并贴入 AI 优化结果。
3. **UI 与系统级交互**：
   - 多面板应用界面（听写、历史、设置）。
   - 沉浸式浮空胶囊窗（Indicator），支持实时麦克风音量波形展示。
   - 依赖 Tauri 实现系统托盘、全局快捷键监听、特定应用黑名单屏蔽。

---

## 2. 当前存在的潜在问题（待优化点）

通过代码审查，以下细节目前尚未完全闭环或存在潜在风险：

1. **录音超时限制未实现**：PRD 中提及“单次按住录音时长上限建议 5 分钟，超时自动截断并提示”，但在 `AudioRecorder` 与 `App.tsx` 的交互中暂未看到硬性的计时器阻断机制。
2. **长文本替换的性能瓶颈**：在 `lib.rs` 的 `replace_with_ai_text` 方法中，使用了 `for _ in 0..original_len { enigo.key_click(Key::Backspace); }` 逐字删除。若用户一次性输入了 500 字，按每次退格间隙 2ms 算，会产生肉眼可见的删除延迟，且期间用户若误触鼠标键盘会导致输入错乱。
3. **撤销功能的可靠性**：PRD 提及“用户可 Ctrl+Z 恢复原文”。因为目前是先 N 次 Backspace 再粘贴，部分编辑器的撤销栈可能会将 Backspace 视为多次操作，导致 Ctrl+Z 无法一键恢复长文本。
4. **SenseVoice 模型管理的固化**：当前下载和加载逻辑在 Rust 后端实现，但如果后续更新模型，缺乏可视化的重新下载或校验机制。

## 3. 下一步开发计划 (Next Steps)

建议按照以下三个阶段逐步推进，您可以指定从哪个阶段或具体任务开始。

### 阶段一：体验打磨与稳定性修复（高优先级）
- [ ] **优化文本替换逻辑**：探索使用 `Shift + 批量左方向键` 选中或操作系统的全选逻辑来替代逐字 Backspace 方案，提升长文本替换速度和撤销体验。
- [ ] **实现录音超时保护**：在 `App.tsx` 或 `AudioRecorder` 中增加计时器，录音超过 5 分钟自动调用 `stopAndProcess()` 并给用户超时的 Toast 提示。
- [ ] **异常与兜底处理**：
  - 增加对麦克风权限被系统拒绝的明确引导（尤其是 macOS）。
  - 处理 AI 接口超时重试机制。
- [ ] **输入法兼容性测试**：在 Windows/Mac 环境下，测试在中文输入法处于“选词状态”时按下快捷键是否会导致输入法卡死，并增加相应的防御逻辑。

### 阶段二：功能拓展与高级特性（中优先级）
- [ ] **场景化 Prompt 一键切换**：在浮空状态窗或主界面顶部，增加一个快速下拉/按钮，允许用户随时在“邮件、笔记、日常聊天”等预设风格间切换。
- [ ] **个人专有词库/热词支持**：允许用户配置常用专业术语，传给 ASR 引擎（如果是云端 API 或支持 prompt 的 Whisper）以提高专业场景识别率。
- [ ] **历史记录功能增强**：支持批量导出、支持通过历史记录再次重试（如果某次网络问题导致润色失败，可点击历史记录进行重新润色）。

### 阶段三：打包发布与工程化（低优先级）
- [ ] **应用图标与 Metadata 更新**：替换默认的 Tauri 图标，配置产品名称和版本号。
- [ ] **打包配置完善**：配置跨平台安装包生成（Windows 的 .msi/.nsis，macOS 的 .dmg），并签署应用证书。
- [ ] **自动更新集成**：利用 Tauri 的 updater 插件实现软件自动检查更新功能。
